# 训练流程

训练流程的目标是让每次实验都能复现。训练脚本不应该依赖手动改代码，而应该从配置文件读取数据路径、模型类型、输入尺寸和训练参数。

## 检测训练

检测训练的输入包括图像、检测标注和数据配置。标注中每个缺陷框都要有类别和坐标。

训练脚本会读取数据配置，加载 YOLO 模型，并在训练集上更新权重。输出包括最佳权重、训练日志、验证集指标和预测可视化图。

判断训练是否正常，可以看四件事：

1. loss 是否下降。
2. 验证集 mAP 是否稳定上升。
3. 每类 Recall 是否合理。
4. 预测图是否和真实缺陷位置大体一致。

如果总 mAP 很高，但稀有类别 Recall 接近 0，说明模型没有真正解决长尾问题。

## 分类训练

分类训练的输入是一组图像和图像级标签。ELPV 可以被整理成二分类、四分类或回归任务。

二分类适合快速判断正常或异常。四分类适合观察缺陷严重程度。回归适合输出连续缺陷分数。

输出包括模型权重、混淆矩阵、F1、AUC 和错误样本列表。正常结果应该能说明模型错在哪些等级上，而不是只给一个准确率。

## 异常检测训练

异常检测训练只使用正常样本。输入是正常训练集和用于调阈值的验证集。输出是异常检测模型、异常分数分布和热力图。

判断结果是否正常，需要同时看正常样本和缺陷样本。如果正常样本分数普遍很高，模型会造成大量误报。如果缺陷样本分数也很低，模型没有学到有效的正常模式。

## 实验记录

每次训练都应该记录模型、数据版本、输入尺寸、随机种子、主要参数和指标。记录的输出可以是 Markdown 表格或 JSON 文件。

这样做的原因很直接：如果某次实验效果更好，必须能知道它为什么更好。如果复现不了，就不能把它当成可靠结论。
