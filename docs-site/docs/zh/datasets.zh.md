# 数据集

本页说明项目会怎样处理光伏缺陷数据。重点不是把数据下载下来就结束，而是让每一步都有可检查的输入和输出。

## PVEL-AD

PVEL-AD 是用于光伏电池片异常检测的 EL 图像数据集。它适合做目标检测，因为标注中包含缺陷框。

输入是原始图像和原始标注。处理后的输出应该是统一格式的数据集，例如 YOLO 标签或 COCO JSON。YOLO 格式适合直接训练 YOLO 类检测器，COCO 格式适合更通用的检测评估工具。

在本项目中，PVEL-AD 用来训练多类别缺陷定位模型。模型需要输出缺陷框、类别和置信度。结果是否正常，主要看每个类别的 AP 和 Recall，而不是只看总 mAP。原因是工业数据经常长尾分布，常见缺陷会把总指标抬高，稀有缺陷却可能仍然漏检。

## ELPV

ELPV 是单体电池片 EL 图像数据集。每张图有一个缺陷概率标签，常见取值是 `0.0`、`0.33`、`0.66` 和 `1.0`。

输入是一张灰度电池片图像和一个概率标签。输出可以有三种形式：

1. 二分类标签：正常或异常。
2. 四级分类标签：四个概率等级。
3. 回归标签：直接预测缺陷概率。

在本项目中，ELPV 还会用于异常检测。训练时只使用正常样本，推理时输出异常分数和异常热力图。结果是否正常，可以看 AUC、正常样本误报率和明显缺陷样本召回率。

## PV-Multi-Defect

PV-Multi-Defect 提供光伏缺陷图像和检测标注，常见组织方式接近 VOC 数据集。

输入是图像和 XML 标注。处理后的输出是统一检测格式。转换时必须检查类别名、图像尺寸、框坐标和空标注。

在本项目中，它用于验证检测训练流程是否能迁移到另一个光伏缺陷数据集。如果同一套转换、训练和评估脚本能处理 PVEL-AD 和 PV-Multi-Defect，说明工程结构没有过度绑定某一个数据源。

## 数据检查

每个数据集进入训练前都要生成检查报告。报告至少包括图像数量、尺寸分布、类别分布、缺失标注、越界框、重复图片和损坏图片。

输入是 `datasets/raw/` 中的原始数据。输出是 Markdown 报告和可视化图片。正常结果应该能回答两个问题：数据能不能被训练脚本读取，以及当前数据分布会不会影响模型评估。
